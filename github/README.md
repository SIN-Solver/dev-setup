# GitHub Guides

Dieser Ordner bündelt unsere internen GitHub-Guides für Teammitglieder.

Die Idee ist einfach:

- bessere Profile
- klarere Teamstruktur
- sauberere Zusammenarbeit
- weniger Chaos zwischen Diskussion, Planung und Umsetzung

## Empfohlene Reihenfolge

| Guide                                                                | Zweck                                                              |
| :------------------------------------------------------------------- | :----------------------------------------------------------------- |
| [`github-profil-readme.md`](./github-profil-readme.md)               | starkes persönliches GitHub-Profil aufbauen                        |
| [`github-organization-readme.md`](./github-organization-readme.md)   | verstehen, was eine GitHub Organization ist und wie wir sie nutzen |
| [`github-project-readme.md`](./github-project-readme.md)             | GitHub Projects als operative Steuerungsfläche nutzen              |
| [`github-discussions-readme.md`](./github-discussions-readme.md)     | Discussions sinnvoll für Ideen, Fragen und Austausch einsetzen     |
| [`github-collaboration-readme.md`](./github-collaboration-readme.md) | Issues, Branches, PRs und Reviews sauber nutzen                    |

## Lernpfad für neue Teammitglieder

1. Erst das **Profil** verstehen und sichtbar machen.
2. Dann die **Organization** verstehen.
3. Danach **Projects** und **Discussions** unterscheiden.
4. Zum Schluss den operativen Flow aus **Issues + PRs + Review** verinnerlichen.

## Ziel

Am Ende soll jedes Teammitglied verstehen:

- wie man auf GitHub professionell wirkt
- wie Teamarbeit in GitHub strukturiert wird
- wie Arbeit von Idee bis Merge sauber geführt wird
